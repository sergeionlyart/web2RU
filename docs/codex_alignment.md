# Codex ↔ Web2RU: практическое сопоставление

## 1) Что Codex умеет (важное для проекта)
- Codex CLI — локальный агент: читает репозиторий, правит файлы, запускает команды.
- Работает лучше, когда есть:
  - воспроизводимая среда,
  - стабильные тесты,
  - AGENTS.md с правилами и командами,
  - skills для повторяющихся задач.

## 2) Какой жизненный цикл задачи считаем “нормальным”
1) Issue-style prompt с критериями приёмки.
2) Plan → маленькие диффы.
3) Прогон quality gates.
4) PR + ревью (особенно security-sensitive части).
5) Merge в main.
6) Release workflow отдельно.

## 3) Где Codex особенно полезен в Web2RU
- быстрое создание “скелета” проекта,
- реализация повторяющихся трансформеров по фикстурам,
- написание unit тестов (особенно на DOM/CSS rewrite),
- рефакторинги с сохранением контрактов,
- подготовка docs.

## 4) Где нужна повышенная осторожность (human-in-the-loop)
- любые изменения, затрагивающие сеть / sandbox / безопасность,
- freeze-js и “0 external requests” валидатор,
- Token Protector и contract переводов,
- live regression по реальным сайтам.

## 5) Рекомендации по паттернам агентской разработки
- Разделяй на роли: Spec → Architect → Implement → Test → Security → Release.
- Тесты и инварианты — главный “guardrail”.
- Live сеть — всегда opt-in.
