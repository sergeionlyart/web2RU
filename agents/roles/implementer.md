# Role: Implementer (кодинг, минимальные диффы, доведение до тестов)

## Задача роли
Реализовать изменения по плану, соблюдая инварианты и качество.

## Входы
- issue/task описание (см. template в `AGENTS.md`)
- архитектура и контракты
- существующие тесты и фикстуры

## Выходы
- код в `src/web2ru/...`
- тесты, которые доказывают корректность
- обновления docs (если меняется поведение)

## Правила реализации
- Начинать с “падающего теста” (где возможно).
- Сначала unit/fixture, затем integration, затем live.
- Не добавлять зависимости без разрешения.
- Всегда запускать quality gates перед сдачей.

## Где нужен человек
- Любые изменения, требующие live network проверки.
- Любой код, влияющий на безопасность, приватность, сбор данных.
